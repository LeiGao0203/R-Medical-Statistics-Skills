# 多维列联表的对数线性模型 (Log-linear Models for Contingency Tables)

> 本章内容改编自 阿越就是我 的《R语言实战医学统计》（https://github.com/ayueme/R_medical_stat），采用 CC BY-SA 4.0 许可证发布。

## 适用场景

**使用对数线性模型的典型场景：**

- 分析三维及以上列联表中多个分类变量之间的关系结构（如病例对照研究中研究暴露因素、基因型与疾病状态的交互作用）
- 检验变量间的条件独立性——在控制某一变量后其余变量是否独立（如控制护理地点后产前护理量与婴儿存活是否独立）
- 检验分层列联表中的同质关联（各层 OR 是否一致），即某交互效应在不同层级下是否稳定
- 需要使用后退法从饱和模型中筛选出最简约的有效模型

**不使用对数线性模型的情况：**

| 你的情况 | 应使用的方法 |
|----------|-------------|
| 仅分析单个二维列联表的独立性 | 卡方检验（`chisq.test()`），对数线性模型也等效 |
| 因变量为连续变量，需控制混杂因素 | 协方差分析（ANCOVA）或多重线性回归 |
| 因变量为二分类变量，需建模预测 | Logistic 回归 |
| 因变量为计数/率，需建模预测 | Poisson 回归或负二项回归 |
| 仅需要计算 OR/HR 的调整值 | Mantel-Haenszel 分层分析 |

**医学研究常见应用：**

- 分层分析中检验各层效应是否同质，避免简单合并产生辛普森悖论（课本例17-1）
- 病例对照研究中评估基因-环境交互作用（课本例17-4）
- 临床试验中比较多种治疗方案的疗效差异（课本例17-3）

## 前置条件

**R 包安装：**

```r
install.packages(c("MASS", "haven", "Crosstabs.Loglinear"))
```

- `MASS` 提供 `loglm()` 函数，用公式形式拟合对数线性模型，输出简洁
- `haven` 用于读取 SPSS 格式数据（`.sav`）
- `Crosstabs.Loglinear` 提供 `LOGLINEAR()` 函数，输出与 SPSS 风格相近的全面结果
- 基础 R 自带 `loglin()` 函数，功能等效但接口较底层

**数据格式要求：**

- 数据应为长格式：每行一个单元格，必须包含分类变量（factor 类型）和频数列（numeric 类型）
- 典型结构如 `人群分组`（病例/对照）、`暴露水平`（暴露/未暴露）、`基因型`（突变/野生）、`频数`

示例（`data17_4` 结构）：

```
##   人群分组 口服避孕药暴露水平 基因型  频数
## 1 病例组   暴露               突变型    25
## 2 病例组   暴露               野生型    84
```

**统计假设：**

1. **多项分布假设**：单元格频数服从多项分布，各观测独立
2. **样本量充足**：期望频数不宜过小（< 1 或 > 20% 单元格 < 5），否则 G² 不可靠
3. **层次模型原则**：若包含高阶交互则必须保留涉及的所有低阶项

## 方法选择决策树

```
你的列联表维度 →
├── 二维列联表（如2×2）→
│   ├── 仅需检验独立性 → 卡方检验即可，对数线性模型等效
│   └── 需计算OR值 → 用 `glm(频数 ~ A * B, family = poisson())` 取系数 `exp()`
├── 三维列联表（如2×2×2）→
│   ├── 想检验某两个变量是否独立（忽略第三个） → 谨慎！可能有辛普森悖论
│   ├── 想检验条件独立性（在Z一定水平下X与Y独立） → 拟合无二阶交互模型
│   ├── 想检验同质关联（各层OR一致） → 用无三阶交互模型 vs 饱和模型
│   └── 想检验三阶交互（基因×环境交互作用） → `step()` 后退筛选
└── 四维及以上 →
    ├── 模型个数爆炸，必须使用后退法（`step()`）自动筛选
    ├── 若需偏关联检验 → 使用 `LOGLINEAR()` 的 Partial Associations 输出
    └── AIC 过小时优先选 AIC 最小的模型，同时检查各残差
```

## 标准工作流

### 步骤1：数据准备与探索

使用 `haven::read_sav()` 读入 SPSS 数据，再用 `haven::as_factor()` 将数值编码转为因子标签。用 `str()` 检查变量类型和频数列。

### 步骤2：拟合饱和模型

以公式 `频数 ~` 拟合饱和模型，`.^2` 表示所有二阶交互，`.^3` 表示所有三阶交互。饱和模型完美拟合数据（G² = 0, df = 0）。

### 步骤3：模型筛选

使用 `step(model, direction = "both")` 基于 AIC 逐步后退筛选，去掉无统计意义的高阶交互项。亦可手动比较简化模型与饱和模型的 `anova()` 结果。

### 步骤4：拟合优度检验

对最终简化模型，查看 `Likelihood Ratio` 的 G² 和 P 值。若 P > 0.05，说明简化模型与饱和模型无显著差异，简化模型可接受。

### 步骤5：结果报告

报告中应包含：最终模型公式、似然比 G²、自由度、P 值、AIC 值。若需报告 OR 值，用 `glm()` 拟合泊松回归取系数 `exp(coef)`。

## 代码示例

### 示例1：三维列联表——辛普森悖论示例

课本例17-1。产前护理量（X）、婴儿存活（Y）、护理地点（Z），2×2×2 表。

```r
# 不顾护理地点直接合并做卡方 → 辛普森悖论！
M <- matrix(c(20,373,6,316), nrow = 2, byrow = TRUE,
            dimnames = list("产前护理量" = c("少","多"),
                            "存活情况" = c("死","活")))
chisq.test(M, correct = FALSE)
## X-squared = 5.2555, df = 1, p-value = 0.02188  # 看似显著

# 甲诊所分层 → 不显著
M1 <- matrix(c(3,176,4,293), nrow = 2, byrow = TRUE,
             dimnames = list("产前护理量" = c("少","多"), "存活情况" = c("死","活")))
chisq.test(M1, correct = FALSE)
## X-squared = 0.083522, df = 1, p-value = 0.7726

# 乙诊所分层 → 不显著
M2 <- matrix(c(17,197,2,23), nrow = 2, byrow = TRUE,
             dimnames = list("产前护理量" = c("少","多"), "存活情况" = c("死","活")))
chisq.test(M2, correct = FALSE)
## X-squared = 9.6186e-05, df = 1, p-value = 0.9922

# loglin 等效于卡方检验
fm <- loglin(M, margin = list(1, 2), fit = TRUE, param = TRUE)
fm$lrt
## [1] 49.84297
1 - pchisq(fm$lrt, fm$df)
## [1] 1.665557e-12
```

### 示例2：二维列联表——性别与高血压

课本例17-2。使用 `MASS::loglm()` 拟合并比较模型。

```r
library(MASS)
library(haven)

data17_2 <- haven::read_sav("datasets/例17-02.sav", encoding = "GBK")
data17_2 <- haven::as_factor(data17_2)

# 不饱和模型（仅主效应，无交互）
f <- loglm(频数 ~ 性别 + 血压, data = data17_2)
f
##                       X^2 df     P(> X^2)
## Likelihood Ratio 49.84297  1 1.665557e-12
## Pearson          50.04632  1 1.501577e-12

# 与饱和模型比较
f1 <- update(f, ~ .^2)
anova(f, f1)
##           Deviance df Delta(Dev) Delta(df) P(> Delta(Dev)
## Model 1   49.84297  1
## Model 2    0.00000  0   49.84297         1              0

# 用 glm 计算 OR 值
f3 <- glm(频数 ~ 性别 * 血压, data = data17_2, family = poisson())
exp(coef(f3)["性别女性:血压高血压"])
## 0.5587329
```

### 示例4：三维列联表——基因×避孕药交互作用

课本例17-4。人群分组、口服避孕药暴露、基因型（FV-Leiden突变），2×2×2 表。

```r
library(MASS)

data17_4 <- haven::read_sav("datasets/例17-04.sav", encoding = "GBK")
data17_4 <- haven::as_factor(data17_4)

# 拟合不同层级的模型
f1 <- loglm(频数 ~ 人群分组 + 口服避孕药暴露水平 + 基因型, data = data17_4)
f2 <- update(f1, ~ .^2)  # 所有二阶交互
f3 <- update(f1, ~ .^3)  # 饱和模型

# 基于 AIC 逐步筛选
f <- step(f3, direction = "both")
## 最终 AIC 最小的模型:
## 频数 ~ 人群分组 + 口服避孕药暴露水平 + 基因型 + 
##      人群分组:口服避孕药暴露水平 + 人群分组:基因型

f
##                         X^2 df  P(> X^2)
## Likelihood Ratio 0.09702652  2 0.9526447
## Pearson          0.09561789  2 0.9533159

# 比较最终模型与饱和模型
anova(f3, f)
##             Deviance df Delta(Dev) Delta(df) P(> Delta(Dev)
## Model 1   0.09702652  2
## Model 2   0.00000000  0 0.09702652         2        0.95264
```

G² = 0.097, P = 0.9526，简化模型可接受。最终模型保留两个二阶交互、无三阶交互，表明避孕药与基因型独立。

## 结果解读指南

**核心输出指标：**

| 指标 | 含义 | 判断标准 |
|------|------|----------|
| Likelihood Ratio G² | 似然比统计量，模型预测与观测值的偏差 | P > 0.05 表示模型拟合可接受 |
| Pearson X² | Pearson 卡方统计量 | 与 G² 互补参考 |
| df | 模型自由度 | 单元格数 - 模型参数数 |
| AIC | 赤池信息准则 | 越小越好，用于模型间比较 |

**K-Way and Higher-Order Effects 解读：**

- **K = 1 行**：检验"所有主效应和高阶效应均为零"。显著（P < 0.05）说明至少需要一个主效应。
- **K = 2 行**：检验"所有二阶及更高交互均为零"。显著说明至少需要一个二阶交互项。
- **K = 3 行**：检验"三阶及更高交互均为零"。显著说明需要三阶交互（仅三维以上有）。

**Partial Associations（偏关联检验）：** 逐个检验每个效应项在排除高阶交互后的独立贡献，用于判断某交互项是否必要。

**Backward Elimination（后退淘汰）：** 从饱和模型开始逐次删除最不显著的项，直到所有保留项均显著或删除后 AIC 不降，最终模型即为最佳模型。

**论文报告模板：**

> 采用对数线性模型分析二维列联表中性别与高血压的关联。结果发现性别×血压交互项有统计学意义（G² = 49.843, df = 1, P < 0.001），说明性别与高血压之间存在关联，女性患高血压的风险低于男性（OR = 0.559）。

## 常见问题与注意事项

**Q1: 对数线性模型和卡方检验有什么区别？**

对于二维列联表，两者等效。卡方检验只能检验独立性，对数线性模型可扩展到三维及以上列联表，分析多变量间的复杂交互结构。

**Q2: 为什么不能简单合并三维表后做卡方？**

可能存在辛普森悖论——分层变量与行/列变量不独立时，合并引入混杂偏倚。例17-1 合并后 P = 0.022（假阳性），分层后均 P > 0.7（实际无关）。

**Q3: 饱和模型 G² = 0 是什么含义？**

饱和模型参数数等于单元格数，完美拟合数据。G² = 0, df = 0 只是出发点，需通过后退法找到简约又拟合良好的不饱和模型。

**Q4: `loglin()` vs `loglm()` vs `LOGLINEAR()` 如何选择？**

- `loglin()`：R 自带，需手动指定 margin 列表，适合编程自动化
- `MASS::loglm()`：公式接口直观，`update(.^2)` 生成二阶交互便捷，推荐日常使用
- `Crosstabs.Loglinear::LOGLINEAR()`：输出最全面，格式匹配 SPSS，适合详细报告

**Q5: AIC 和 BIC 怎么选？**

AIC 倾向复杂模型，BIC 惩罚更重、倾向简化模型。医学统计中两者均可，报告时说明使用的准则。

**Q6: 后退法删除交互项时，主效应要保留吗？**

必须保留。对数线性模型是层次模型，`step()` 自动遵守层次原则，不会删除低阶项。

**Q7: 单元格期望频数很小时怎么办？**

若期望频数 < 1 或 > 20% 单元格 < 5，G² 卡方近似不可靠。可：① 合并某些水平；② 使用精确检验；③ 增加样本量。

**Q8: 模型输出中"无二阶交互模型"和"条件独立模型"有何区别？**

这是模型的不同简化层级：无二阶交互模型仅含主效应（完全独立）；条件独立模型含部分二阶交互（在给定某变量时其余变量独立）；同质关联模型含所有二阶交互但无三阶交互（检验交互效应在各层一致）；饱和模型含所有交互完美拟合。
